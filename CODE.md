# Code

## Purpose
Pi extension package for Codex/GPT service-tier controls.

## Package
- `package.json`: npm/pi package manifest with Pi/Codex/GPT/OpenAI/service-tier keywords. Pi loads `./extensions` via `package.json#pi.extensions`. Scripts include `test` and `test:apply-patch` for Bun test harness.
- Published files: `extensions/`, `README.md`, `LICENSE`.
- Runtime dependencies: none.
- Peer deps require Pi packages; dev deps provide TypeScript/Biome tooling.

## Extension
- `extensions/lovely-codex/index.ts`: Pi entrypoint. Owns in-memory config by scope (`global`/`project`), refreshes both on `session_start`, and registers `/codex`, GPT mode hooks, and `apply_patch`. Hooks merge scopes on read, with project overriding global. Effective `applyPatchMode` mutates active tools at runtime: `disabled` removes `apply_patch`; `enabled` enables `apply_patch`; both non-replace modes restore `edit`/`write` only if they were active at session start; `replace-edit` enables `apply_patch` and removes active `edit`/`write`. Invalid config reload fails closed to default and clears status. Non-default mode sets a `lovely-codex` status indicator (`đźŹŽď¸Ź`); default clears it.
- `extensions/lovely-codex/config.ts`: TypeBox-validated config helpers. Provides config paths, scoped loading, merge helper, and file read/write. Config shape: `{ "gptMode"?: "default" | "fast" | "fast-codex", "applyPatchMode"?: "disabled" | "enabled" | "replace-edit" }`; omitted settings mean unset; default fallbacks are `gptMode=default` and `applyPatchMode=enabled`.
- `extensions/lovely-codex/command.ts`: `/codex` command. Takes no args. TUI shows tabbed `User` (global) and `Workspace` (project) scopes with two setting rows: `GPT mode` cycling through `unset`, `default`, `fast`, `fast-codex`; and `apply_patch` cycling through `unset`, `disabled`, `enabled`, `replace-edit`. Notes beside each row show Workspace overrides, User inheritance, or default fallback. Writes only active scope and updates in-memory config + active tools. Bad scoped config shows a warning and only that scope starts empty.
- `extensions/lovely-codex/gpt-mode.ts`: request hooks for OpenAI GPT models (`provider` `openai`/`openai-codex`, model id `gpt-*`). Reads current mode from the config closure. `before_provider_request` injects `service_tier: "priority"` only when enabled; `default` omits the field, `fast` prioritizes both providers, `fast-codex` only `openai-codex`. `message_end` checks current mode and adjusts priority cost for `openai-codex` assistant messages.
- `extensions/lovely-codex/apply-patch.ts`: `apply_patch` tool wrapper with schema `{ input: string }`. Prompt text mirrors Codex apply_patch guidance while omitting freeform-specific wording: use it to edit files, allow multi-file/multi-hunk patches, require the envelope, file-operation headers, `+` prefixes, and relative paths. Parses touched paths from apply-patch envelope, best-effort snapshots files before/after, shells out to `codex --codex-run-as-apply-patch <input>` in `ctx.cwd`, returns concatenated `stdout+stderr` as tool text on success, throws combined output plus partial-change diff on non-zero exit, and keeps raw `{ exitCode, stdout, stderr, output }` plus edit-style details (`diff`, unified `patch`, `firstChangedLine`). TUI call line shows touched filenames highlighted like `edit`; result renders line-numbered diffs via Pi `renderDiff`, prefixing filename headers only for multi-file diffs.

## Apply patch tests
- `tests/apply-patch/runners.ts`: black-box runner abstraction. Includes direct Codex runner plus Pi wrapper runner using `runCodexApplyPatch()` from extension code.
- `tests/apply-patch/fs.ts`: temp-dir, fixture copy, and recursive filesystem snapshot helpers.
- `tests/apply-patch/scenario.test.ts`: runs copied upstream fixture scenarios against each runner, comparing final filesystem state only.
- `tests/apply-patch/cli.test.ts`: explicit stdout/stderr/exit-code cases for core CLI semantics against each runner.
- `tests/apply-patch/fixtures/scenarios/`: copied Codex fixture corpus, unchanged layout (`input/`, `expected/`, `patch.txt`).

## Tooling
- `tsconfig.json`: strict TypeScript config for `extensions/` and `tests/`.
- `biome.json`: formatter/linter config matching adjacent Pi Lovely packages.
- `bun.lock`: local dependency lock generated by `bun install`.
